1.   命名规范

文件命名：文件应该以标志文件属性的前缀+标志该文件功能的英文词组或词组缩写构成。前缀采用小写，词组中每个词的首字母应该大写，某写专有名词或缩写按照习惯用法确定大小写，如:MDI、ActivtX等。

2.   书写格式

缩进：缩进采用4个字符。

程序段: 程序段之间、函数、过程之间应该有两行空行。

语句：为保证代码的易读性，要求完整地将语句写出，并且不要在同一行写完一条语句。

正确的写法：

If … Then

End if

Do While …

Loop

For 变量=初始值 To 终止值

Next 变量。

错误的写法：

If … Then …或 If … Then…End If

If … Then

EndI

For 变量=初始值 To 终止值

Next。

3.   代码要求

一般原则：

1．写清晰的代码：比如，写代码应该注意程序动作的连续性，不要在某处对一个变量赋值，又在100行代码以后才使用它等等。

2．写简单的代码：比如，为提高效率或为了使代码看来短一些而采用一些技巧等等。

3．时刻注意系统开销：比如，不要保留不使用的变量和代码；不要为数组定义超过使用需要的空间；在同一窗体中若使用多个同类型的对象，最好采用对象数组等等。

4．保证系统的模块化：比如，尽量少使用全局变量；在使用全局变量时应尽量避免在多处都可写的情况发生；尽量少使用Goto语句。

5．时刻注意对浮点数据类型的处理：比如，若浮点类型变量dblA,dblB进行比较时，采用 If dblA-dblB=0 Then … End If 的判断是错误的。

6.使用的变量都应该预先定义，若非特殊情况，应该准确地定义出变量的类型。

正确的写法：

Dim strSql As String

错误的写法：

Dim strSql

函数和过程的使用：当某段代码的使用次数超过一次而又具有一定的整体性时，应该采用函数或过程的方式来处理；在窗体中出现的响应事件的函数的代码长度应该限制在500行以内，否则使用过程或函数来实现；当某个模块中的代码太长时，应尽量根据语句的意群分割为过程或函数。

程序注释：程序的注释包括程序头说明及代码注释。

程序头说明：说明本程序的作者；代码开始时间、完成时间、需求或设计文档编号；需求变动的开始时间、完成时间、相应需求或设计文档编号；调用的外部函数或模块；被哪些模块调用。

代码注释：当代码涉及到业务时应该加注释；在函数和过程及程序段前应该加注释表明函数或过程或程序段的功能，入口参数的意义，返回值的意义；对于全局变量，模块级变量应该加注释；对重要的局部变量也应该加注释。
